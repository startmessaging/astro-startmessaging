O disparo de mensagens promocionais em massa é uma das ferramentas mais poderosas para impulsionar vendas rápidas e engajar clientes no mercado brasileiro. A altíssima taxa de abertura e o uso cotidiano do aplicativo criam uma oportunidade valiosa para varejistas e empresas de serviços. No entanto, o envio indiscriminado de spam gera punições e o bloqueio de números. Entender a diferença entre ferramentas informais e o canal oficial de disparo whatsapp em massa (WhatsApp Business API) é essencial para realizar campanhas de marketing seguras. Neste guia, apresentamos as principais diretrizes técnicas e de conformidade para proteger o número de atendimento de sua marca.

A API do StartMessaging para WhatsApp está atualmente em fase de desenvolvimento interno de novas funções. Enquanto nossa equipe de engenharia valida os endpoints definitivos de produção para o ambiente em nuvem, que serão lançados em breve, você já pode desenvolver e testar suas lógicas de disparo utilizando a rota provisória `/v1/messages`. O acesso técnico de autorização é feito de forma simplificada por meio do cabeçalho `X-API-Key`.

## Disparos via API Oficial vs. Ferramentas Não-Oficiais: O Risco de Banimento

Muitos gestores de marketing buscam alternativas de baixo custo para realizar disparos em massa, recorrendo a softwares ou extensões de navegador não oficiais. Essas ferramentas funcionam por meio do espelhamento do aplicativo convencional de celular (WhatsApp Web), enviando mensagens de forma sequencial e simulando a digitação humana de forma automatizada.

O uso dessas ferramentas traz sérios riscos para a sua operação comercial. O sistema de segurança do WhatsApp monitora o comportamento da conta. Padrões de envio rápidos e a ausência de interações de volta ativam alertas de comportamento robótico. O resultado é o banimento definitivo do número de telefone em poucas horas de disparo, sem possibilidade de recuperação do histórico de clientes e dos contatos acumulados na conta.

A API Oficial do WhatsApp Business Platform (WhatsApp Broadcast) é o único método seguro reconhecido pela Meta. Ela funciona por meio de conexões diretas entre servidores, eliminando a dependência de celulares conectados e permitindo o envio de alto volume de mensagens promocionais homologadas. Essa infraestrutura oficial garante taxas de entrega elevadas e segurança técnica para a sua empresa.

## O Sistema de Qualidade de Número (Quality Rating) da Meta

A Meta adota um sistema dinâmico para monitorar a reputação das contas corporativas que utilizam a API do WhatsApp. O chamado **Quality Rating** (Classificação de Qualidade) é exibido no seu painel de controle de negócios e avalia a recepção das suas mensagens pelos destinatários em um intervalo de até sete dias.

A classificação de qualidade é dividida em três níveis principais:

* **Verde (Alta Qualidade):** Indica que a sua empresa segue as melhores práticas de envio, com mensagens relevantes e baixo volume de reclamações ou bloqueios por parte dos destinatários.
* **Amarela (Média Qualidade):** Sinaliza um aumento no número de reportes de spam ou taxas de rejeição. É um alerta para revisar a segmentação das suas listas de contatos e diminuir a frequência dos disparos.
* **Vermelha (Baixa Qualidade):** Indica que o número de telefone está recebendo um alto índice de reclamações. Se o número permanecer nessa classificação por muito tempo, a Meta reduzirá o limite diário de mensagens ou suspenderá temporariamente o envio de campanhas.

Para evitar a perda de reputação do seu número de atendimento, a empresa deve manter a qualidade sob controle. Oferecer campanhas segmentadas baseadas em preferências declaradas de clientes é a melhor alternativa para evitar que os destinatários marquem suas mensagens como conteúdo indesejado.

## Modelos de Mensagem (Templates) Aprovados: Estrutura e Envio

Para iniciar conversas com clientes (mensagens ativas), a API do WhatsApp exige o uso de modelos de mensagem (Message Templates) aprovados previamente. Esses modelos garantem que a empresa siga as políticas de comércio do canal, impedindo o envio de comunicações invasivas ou de spam.

Abaixo, apresentamos o exemplo de código em Node.js utilizando Axios para realizar um disparo de template ativo com o endpoint provisório do StartMessaging:

```javascript
const axios = require('axios');

/**
 * Envia uma mensagem ativa utilizando um template promocional homologado no WhatsApp.
 * 
 * @param {string} toPhone - Telefone do cliente final no formato internacional (ex: "5511977777777")
 * @param {string} clientName - Nome personalizado do cliente para preenchimento de variável
 * @param {string} couponCode - Código do cupom de desconto gerado pelo sistema
 */
async function dispararCampanhaDesconto(toPhone, clientName, couponCode) {
  // TODO: Substituir pelo endpoint de produção definitivo assim que a API do WhatsApp for disponibilizada
  const apiURL = 'https://api.startmessaging.com/v1/messages';

  const payload = {
    to: toPhone,
    type: 'template',
    template: {
      name: 'promocao_fim_de_semana',
      language: {
        code: 'pt_BR'
      },
      components: [
        {
          type: 'body',
          parameters: [
            { type: 'text', text: clientName },
            { type: 'text', text: couponCode }
          ]
        }
      ]
    }
  };

  try {
    const response = await axios.post(apiURL, payload, {
      headers: {
        'X-API-Key': process.env.STARTMESSAGING_API_KEY,
        'Content-Type': 'application/json'
      }
    });
    console.log(`Mensagem promocional enviada com sucesso para ${clientName}. ID: ${response.data.messageId}`);
    return response.data;
  } catch (err) {
    console.error('Falha no envio do template de broadcast:', err.response ? err.response.data : err.message);
    throw err;
  }
}

// Execução de envio para um cliente da lista
dispararCampanhaDesconto('5511977777777', 'Ana Silva', 'DESCONTO10');
```

Este script estrutura a chamada de rede repassando o identificador do template desejado. Ele injeta as variáveis dinâmicas de nome e código do cupom, enviando a requisição com autenticação segura pelo cabeçalho `X-API-Key`.

## Como Construir uma Lista de Contatos Válida (Opt-in)

O sucesso das suas campanhas de disparos em massa depende da qualidade da sua lista de contatos. Comprar bases de dados prontas de terceiros ou raspar números na internet são práticas ilegais que violam a LGPD e resultam no bloqueio rápido do número de atendimento da empresa.

As principais estratégias legítimas para captação de opt-in incluem:

* **Integração no Fluxo de Compra (Checkout):** Inclua uma caixa de seleção separada no formulário de compra (ex: "Desejo receber cupons e atualizações de produtos via WhatsApp").
* **Pixel de Captura em Landing Pages:** Ofereça cupons de desconto imediatos na primeira compra em troca do preenchimento voluntário do número de WhatsApp em uma página de captura.
* **Uso de QR Code em Lojas Físicas:** Disponibilize QR Codes em locais visíveis na loja física (como no caixa ou em displays de mesa), direcionando o cliente para iniciar uma conversa no WhatsApp e receber ofertas exclusivas automaticamente.

## Melhores Horários e Boas Práticas para Envios

O consumidor brasileiro preza pela conveniência do contato, mas se incomoda com abordagens comerciais invasivas fora do horário comercial. Definir regras claras de agendamento em seu sistema evita taxas altas de bloqueio.

As boas práticas recomendadas para envios comerciais incluem:

* **Evitar Horários Extremos:** Nunca envie disparos de marketing antes das 9h ou após as 20h. Além de gerarem reclamações, mensagens enviadas tarde da noite apresentam taxas menores de conversão e podem irritar o cliente.
* **Controlar a Frequência Semanal:** O limite recomendado para disparos de novidades ou promoções gerais é de 1 a 2 contatos por semana para um mesmo número de cliente. Ultrapassar esse limite desgasta o relacionamento com a marca e aumenta os reportes de spam.
* **Oferecer Opção Clara de Opt-out:** Toda mensagem promocional deve conter um comando simples e claro para descadastro de forma visível (ex: "Para parar de receber nossas mensagens, responda com PARAR"). Processar essas exclusões de forma automatizada em seu sistema é essencial para manter a conformidade da conta.

## Perguntas Frequentes

**Q: Qual o custo cobrado pela Meta para o disparo de mensagens promocionais (Marketing)?**

A: Diferente do envio de mensagens de atendimento de suporte (Service), a Meta cobra uma tarifa diferenciada para conversas da categoria "Marketing" (envio de promoções, ofertas e novidades). No Brasil, o valor de cada conversa de marketing iniciada pela empresa varia entre R$ 0,30 e R$ 0,40, dependendo da cotação e de ajustes na política de preços da Meta.

**Q: Posso enviar cupons e links de pagamento Pix em disparos de marketing?**

A: Sim. Você pode criar templates promocionais que incluam botões interativos direcionando o cliente para links de pagamento (como checkout do Paystack ou páginas de ofertas), além de disponibilizar códigos de desconto dinâmicos associados a variáveis em suas mensagens.

**Q: Existe um limite máximo de envios diários de disparos em massa?**

A: Sim, conforme o nível (tier) de qualidade do seu número de WhatsApp corporativo. Contas novas iniciam no limite de 1.000 destinatários únicos por dia. A Meta aumenta esse limite para 10.000, 100.000 e disparos ilimitados à medida que a qualidade da conta se mantém alta e o volume de envios aumenta.

**Q: É necessário que o cliente salve o número da minha empresa na agenda para receber o disparo?**

A: Não. Através da API oficial do WhatsApp Business, as mensagens promocionais serão entregues normalmente na caixa de entrada do destinatário, mesmo que ele não possua o número da sua empresa cadastrado nos contatos do celular.
